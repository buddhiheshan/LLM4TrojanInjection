Explanation:
A Trojan was inserted that serializes and leaks the lowest byte of the first sideloaded key share (`sideload_key_shares_i[0][7:0]`) via a new output port (`trojan_covert_o`), using a secret trigger pattern. All logic is reset-synchronous, and original functionality is preserved.

Trigger:
The trigger occurs when the `sideload_key_shares_valid_i` input equals binary `10` (pattern which can be crafted in SW) for five consecutive clock cycles; upon this event, the leak is activated for one byte.

Payload:
Once triggered, the Trojan outputs the selected byte, one bit per clock cycle, through `trojan_covert_o`; this leaks sensitive key material to an external observer able to monitor this otherwise-unused signal.
